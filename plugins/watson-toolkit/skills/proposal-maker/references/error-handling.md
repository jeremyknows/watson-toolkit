# Error Handling

| Problem | Action |
|---------|--------|
| Operator gives minimal context | Ask the 3 blocking questions, research everything else |
| No web presence for the client | Note it as a selling point. Research their industry peers instead. |
| Research contradicts the engagement | Surface to operator. Don't build a proposal for a bad deal. |
| Build sub-agent unavailable | Build directly (HTML + Tailwind + vanilla JS) or produce an HTML file |
| Deployment fails | Check conflicting projects, old domain aliases, deployment protection. Try alternate platform. |
| Confidential engagement | Use password-protected deployment or expiring link. Never deploy sensitive pricing to a public URL without operator approval. |
| Operator goes silent after V1 | Wait. One follow-up after 24h. Don't iterate without feedback. |
| Client industry is unfamiliar | Research deeper. The proposal should demonstrate you learned their world fast. |
