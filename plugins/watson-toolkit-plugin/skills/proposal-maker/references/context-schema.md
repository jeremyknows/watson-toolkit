# Client Context Schema

Save alongside your project files as `CONTEXT.md`. This is the single source of truth for the engagement.

```markdown
# [Client Name] - Client Project Context

## Overview
[One paragraph: who they are, what they do, why they're a prospect]

## Client Details
- **Business name:** 
- **Location:** 
- **Industry:** 
- **Website:** [URL or "none"]
- **Social:** [links or "minimal"]
- **Business size:** [revenue range, team size, or "unknown"]
- **Decision maker(s):** [name(s) and role(s)]

## The Engagement
- **What we're proposing:** [specific deliverables/services]
- **Engagement model:** [one-time / retainer / fractional / advisory]
- **Pricing:** [proposed or "TBD - research needed"]
- **Timeline:** [expected duration]

## Their Current State
- **What's working:** 
- **What's broken/missing:** 
- **Current pain points:** 

## Target Market
1. [Primary audience]
2. [Secondary audience]
3. [Tertiary if applicable]

## Competition
- [Competitor 1 - one line description]
- [Competitor 2 - one line description]
- [Competitor 3 if applicable]

## Design Direction
- **Reference sites:** [URLs or "none provided"]
- **Brand colors:** [if known]
- **Vibe:** [e.g., "premium/dark", "clean/minimal", "warm/friendly"]

## Notes
[Anything else relevant: meeting notes, relationship context, red flags, opportunities]
```

## Additional Research Files

Save alongside CONTEXT.md in the same folder:
- `competitor-analysis.md` - Detailed competitive teardown
- `market-research.md` - Market sizing and trends
- `pricing-benchmarks.md` - Real pricing data from research
