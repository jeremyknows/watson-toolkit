# Brief Quality Checklist

**Read when:** Stage 7, before filing or posting any brief.

Run through before you hit send. A brief that fails 2+ checks should be revised or flagged with explicit caveats.

## Completeness

- [ ] **Question answered directly** — the Answer section leads with a conclusion, not a summary
- [ ] **Confidence level assigned** — High / Medium / Low with a one-sentence reason
- [ ] **All 7 stages ran** (or fast-path explicitly declared at the top)
- [ ] **Stage 6 counterargument present** — not skipped, not rubber-stamped

## Evidence Quality

- [ ] **No unsourced critical claims** — every skeleton point has at least one citation
- [ ] **Not all sources from the same community** — groupthink risk flagged if so
- [ ] **Source confidence noted** — primary / secondhand / opinion for each key finding
- [ ] **Contradicting evidence addressed** — ❌ findings updated the hypothesis or were explained

## Actionability

- [ ] **Decision Recommendation is specific** — Watch / Use now / Act on / Archive + concrete next step
- [ ] **Audience clear** — who is making the decision this brief supports?
- [ ] **Time horizon noted** — is this brief time-sensitive? Does it have a decay date?

## Format

- [ ] **Short brief (< 300 words):** replied inline — no file needed
- [ ] **Long brief (> 300 words):** written to `docs/research/<slug>-YYYY-MM-DD.md`, linked inline
- [ ] **Research thread:** posted via `message(action=thread-reply)` into the correct thread

## Confidence Calibration Reference

| Level | When to use |
|---|---|
| **High** | Multiple independent primary sources, weak counterargument, no major unknowns |
| **Medium** | Some primary sources, counterargument has merit, 1–2 unknowns |
| **Low** | Mostly secondary sources, strong counterargument, significant unknowns |

Low confidence ≠ don't ship. It means flag it so the decision-maker knows how much weight to put on it.
